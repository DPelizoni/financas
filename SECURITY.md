# Política de Segurança

A segurança do projeto é prioridade.

## Reportar vulnerabilidades

Se você identificar uma vulnerabilidade, não abra issue pública com detalhes sensíveis.

Envie um relato contendo:

- Descrição do problema
- Passo a passo para reproduzir
- Impacto potencial
- Sugestão de correção (se houver)

Até que um canal privado dedicado seja definido, utilize o fluxo de contato direto dos mantenedores do projeto.

## Boas práticas para contribuidores

- Nunca versionar arquivos `.env` reais.
- Não incluir tokens, chaves de API ou senhas em código, commits ou issues.
- Revisar logs e prints antes de compartilhar.
- Preferir placeholders em exemplos (`.env.example`).
